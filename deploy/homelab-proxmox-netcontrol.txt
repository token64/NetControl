Subir la copia local (Windows / XAMPP) al CT o VM
-------------------------------------------------
- **Local = fuente de verdad → Proxmox (VM panel):** desde la raíz del repo:
    .\deploy\sync-to-proxmox-netcontrol.ps1
  Equivale a `sync-local-to-remote.ps1` con host por defecto 192.168.13.50 (VM «NetControls»).
  Requiere `scp`/`ssh` sin contraseña en cada paso (clave SSH) o aceptar los prompts.
- Misma herramienta con IP/usuario a mano:
    .\deploy\sync-local-to-remote.ps1 -RemoteHost IP_DEL_CT_O_VM -RemoteUser jose
- Recomendado a largo plazo: en tu PC `git commit` + `git push`; en el servidor `git pull` (o server-git-pull.sh) o `.\deploy\proxmox-vm-guest-git-deploy.ps1` (solo lo que esté en GitHub).
- Por defecto NO pisa `panel/config.php` del servidor (usá `-IncludePanelConfig` solo si querés copiar también el tuyo).
  Requiere sudo en el remoto para extraer bajo `/var/www/NetControl`.

Actualizar código del CT sin SSH al .73
------------------------------
PVE 9.x no permite "ejecutar bash dentro del LXC" por la API HTTP. Opciones:
- Consola web: Nodo proxmox -> Shell -> pegar el comando que genera:
    deploy\pve-sync-netcontrol.ps1   (usa tu proxmox.credentials.json y encuentra vmid por IP)
- O SSH al hipervisor (.215) y el mismo "pct exec <vmid> -- bash -lc '...'"

VM Linux con qemu-guest-agent (ej. 109 «NetControls», IP .50): sí se puede por API HTTP
usando agent/exec + agent/exec-status. Desde tu PC (PowerShell), con el mismo
proxmox.credentials.json:
    .\deploy\proxmox-vm-guest-git-deploy.ps1
  Parámetros típicos: -Node proxmox -Vmid 109 -RepoPath /var/www/NetControl
  El token de API necesita permisos de agente en esa VM; git corre en el invitado
  (root vía guest agent); si el repo no es de root, ya usa git -c safe.directory=...

Nota: Las IPs pueden cambiar; ajustá según tu red.

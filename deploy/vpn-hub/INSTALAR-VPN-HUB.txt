VPN hub — VM Ubuntu + WireGuard (referencia de despliegue)
===========================================================

1) Instalá Ubuntu Server 24.04 en una VM (ISO desde el hipervisor o imagen cloud).
   - Activá OpenSSH.
   - IP fija recomendada en tu LAN (ej. rango que uses: 192.168.x.x, 10.x.x.x, etc.).

2) Copiá los archivos a la VM (desde Windows, en la carpeta del repo NetControl).
   Mejor (evita doble carpeta vpn-hub/vpn-hub y CRLF si podés):
     scp -r deploy\vpn-hub\* usuario@IP_VM:~/vpn-hub/
   Si usaste «scp -r deploy/vpn-hub …:~/vpn-hub» y quedó todo adentro de ~/vpn-hub/vpn-hub,
     cd ~/vpn-hub/vpn-hub

3) En la VM (en la carpeta donde están los .sh):
   Si al ejecutar ves «set: pipefail» o «invalid option», los scripts traen finales de línea Windows. Corregí:
     sudo apt install -y dos2unix && dos2unix *.sh
   o:  sed -i 's/\r$//' *.sh
   Luego:
     sudo bash ./prepare-wireguard-hub.sh

   Anotá la «clave pública del servidor» que imprime al final.

4) MikroTik borde (tu router WISP)
   - Resumen: NAT dst-nat UDP 51820 WAN → IP LAN de la VM hub; DHCP lease MAC→IP opcional.
   - Detalle y comandos: MIKROTIK-BORDE.txt (misma carpeta).

5) Firewall general
   - Reenvío UDP 51820 hacia la VM; UFW en Ubuntu según prepare-wireguard-hub.sh.

6) MikroTik como cliente WireGuard (RouterOS 7)
   - Interface → WireGuard → + : Name wg-netcontrol, Private Key (generado en el MK: /interface wireguard print keys después de crear).
   - Peer: Public Key = la del servidor (paso 3); Endpoint = IP pública o DDNS de tu casa + puerto 51820; AllowedIPs puede ser 0.0.0.0/0 (ruteo completo) o solo 10.64.0.1/32 según diseño.
   - IP → Addresses: asigná en wg-netcontrol la IP del túnel (ej. 10.64.0.16/24) coherente con lo que registrarás en el hub.

7) Registrar el peer en el servidor
   sudo bash wg-server-add-peer.sh nombre_router 'CLAVE_PUBLICA_DEL_MIKROTIK' 10.64.0.16/32

8) NetControl
   Routers → IP/host = 10.64.0.16 (o la que usó el MK) + usuario/contraseña API.

9) Panel web PHP (alta de peers desde el navegador)
   Carpeta en el repo: deploy/vpn-hub/web-panel/
   - Opción rápida (en la VM, con todos los archivos de vpn-hub en ~/vpn-hub, incl. web-panel/):
       cd ~/vpn-hub && sudo bash ./install-web-panel-apache.sh
     Eso instala Apache+PHP, copia web-panel a /var/www/html/vpn-panel y arranca el servicio.
   - Manual: Instalá Apache y PHP (en la VM hub):
     sudo apt update && sudo apt install -y apache2 libapache2-mod-php php
   - Copiá la carpeta web-panel al servidor web (ej. desde tu PC, ajustá usuario e IP LAN de la VM):
     scp -r deploy\vpn-hub\web-panel usuario@IP_VM:/tmp/web-panel
     ssh usuario@IP_VM
     sudo rm -rf /var/www/html/vpn-panel && sudo mv /tmp/web-panel /var/www/html/vpn-panel
     sudo chown -R root:www-data /var/www/html/vpn-panel
     sudo chmod 640 /var/www/html/vpn-panel/config.php 2>/dev/null; true
   - Configuración:
     sudo cp /var/www/html/vpn-panel/config.example.php /var/www/html/vpn-panel/config.php
     sudo nano /var/www/html/vpn-panel/config.php
       · NC_VPN_PANEL_PASSWORD = contraseña larga
       · WG_ADD_PEER_SCRIPT = ruta absoluta a wg-server-add-peer.sh (ej. /home/jose/vpn-hub/wg-server-add-peer.sh)
   - Sudo solo para el script y wg show (ajustá la ruta del .sh):
     sudo install -m 440 /dev/null /etc/sudoers.d/nc-vpn-panel
     sudo nano /etc/sudoers.d/nc-vpn-panel
       www-data ALL=(root) NOPASSWD: /bin/bash /home/jose/vpn-hub/wg-server-add-peer.sh
       www-data ALL=(root) NOPASSWD: /usr/bin/wg show wg0
     (Si wg está en otra ruta: which wg)
   - Si usás UFW y querés HTTP desde la LAN:
     sudo ufw allow from 192.168.13.0/24 to any port 80 proto tcp
     (Ajustá el /24 a tu red; en producción conviene HTTPS y restricción por IP.)
   - Reiniciá Apache: sudo systemctl reload apache2
   - Abrí en el navegador (reemplazá por la IP LAN real de la VM):
     http://IP_DE_LA_VM/vpn-panel/
   - La primera vez: entrá con la contraseña de config.php. NO publiques el panel a Internet sin HTTPS y sin endurecer acceso.
   - Si el login dice «Contraseña incorrecta»: la cadena debe coincidir con NC_VPN_PANEL_PASSWORD
     (mayúsculas; sin espacios de más en config.php ni al pegar).
     Comprobar: php -r "require '/var/www/html/vpn-panel/config.php'; echo NC_VPN_PANEL_PASSWORD, PHP_EOL;"
   - Si la contraseña es correcta pero volvés al login sin error: probá ventana privada,
     borrá cookies del sitio, o actualizá index.php (sesión SameSite Lax + redirect 303).
     Diagnóstico en la VM: curl -sS -c /tmp/ncj -b /tmp/ncj -d 'login=1&password=TU_CLAVE' -X POST 'http://127.0.0.1/vpn-panel/' -D - -o /tmp/out.html | head -20
     (debe aparecer HTTP/1.1 303 y en /tmp/out.html el formulario de peers, no solo el login).
   - Para traer cambios del panel desde Git (ej. después de git pull en ~/NetControl):
     sudo cp ~/NetControl/deploy/vpn-hub/web-panel/index.php /var/www/html/vpn-panel/index.php

Nota: Si ya tenés otra VPN (p. ej. la del proveedor con OpenVPN tcp 1194), mantené cada túnel con su red; este hub NetControl usa 10.64.0.0/24 aparte.
   Si necesitás OpenVPN tcp por compatibilidad exacta, armá servidor OVPN aparte o pedí script dedicado.

Archivos
--------
- prepare-wireguard-hub.sh  — instalación hub
- wg-server-add-peer.sh     — alta de cada MikroTik / cliente
- web-panel/                — panel PHP (nombre, descripción, clave pública, IP túnel)
- install-web-panel-apache.sh — instala Apache+PHP y publica /vpn-panel/ en la VM
- MIKROTIK-BORDE.txt        — NAT 51820, DHCP, ruta 10.64.0.0/24 en el router central

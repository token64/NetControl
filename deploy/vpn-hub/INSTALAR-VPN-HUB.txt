VPN hub — VM Ubuntu + WireGuard (referencia de despliegue)
===========================================================

1) Instalá Ubuntu Server 24.04 en una VM (ISO desde el hipervisor o imagen cloud).
   - Activá OpenSSH.
   - IP fija recomendada en tu LAN (ej. rango que uses: 192.168.x.x, 10.x.x.x, etc.).

2) Copiá esta carpeta a la VM (desde tu PC con la IP que tomó la VM):
     scp -r deploy/vpn-hub usuario@IP_VM:~

3) En la VM:
     cd ~/vpn-hub
     sudo bash prepare-wireguard-hub.sh

   Anotá la «clave pública del servidor» que imprime al final.

4) MikroTik borde (tu router WISP)
   - Resumen: NAT dst-nat UDP 51820 WAN → IP LAN de la VM hub; DHCP lease MAC→IP opcional.
   - Detalle y comandos: MIKROTIK-BORDE.txt (misma carpeta).

5) Firewall general
   - Reenvío UDP 51820 hacia la VM; UFW en Ubuntu según prepare-wireguard-hub.sh.

6) MikroTik como cliente WireGuard (RouterOS 7)
   - Interface → WireGuard → + : Name wg-netcontrol, Private Key (generado en el MK: /interface wireguard print keys después de crear).
   - Peer: Public Key = la del servidor (paso 3); Endpoint = IP pública o DDNS de tu casa + puerto 51820; AllowedIPs puede ser 0.0.0.0/0 (ruteo completo) o solo 10.64.0.1/32 según diseño.
   - IP → Addresses: asigná en wg-netcontrol la IP del túnel (ej. 10.64.0.16/24) coherente con lo que registrarás en el hub.

7) Registrar el peer en el servidor
   sudo bash wg-server-add-peer.sh nombre_router 'CLAVE_PUBLICA_DEL_MIKROTIK' 10.64.0.16/32

8) NetControl
   Routers → IP/host = 10.64.0.16 (o la que usó el MK) + usuario/contraseña API.

Nota: Si ya tenés otra VPN (p. ej. la del proveedor con OpenVPN tcp 1194), mantené cada túnel con su red; este hub NetControl usa 10.64.0.0/24 aparte.
   Si necesitás OpenVPN tcp por compatibilidad exacta, armá servidor OVPN aparte o pedí script dedicado.

Archivos
--------
- prepare-wireguard-hub.sh  — instalación hub
- wg-server-add-peer.sh     — alta de cada MikroTik / cliente
- MIKROTIK-BORDE.txt        — NAT 51820, DHCP, ruta 10.64.0.0/24 en el router central
